# Hosted Deployment Guide

This project is designed so your public API server orchestrates jobs while a third-party provider (for example RunComfy) executes the actual ComfyUI workflow.

## 1) Required API server env vars

- `HDRI_PUBLIC_BASE_URL` - public HTTPS URL for signed file links.
- `HDRI_SIGNING_SECRET` - strong random secret for signed URL generation.
- `HDRI_DB_PATH` - SQLite path for jobs/accounts/token ledger.
- `HDRI_REQUIRE_API_KEY=1` - enforce Bearer auth for generation endpoints.
- `HDRI_ADMIN_TOKEN` - required to call `POST /v1/accounts`.

Recommended:

- `HDRI_MAX_ACTIVE_JOBS_PER_ACCOUNT=2` - per-account abuse/concurrency guard.
- `HDRI_STALE_REAPER_INTERVAL_S=30` - stale-job scan interval.
- `HDRI_STALE_JOB_TIMEOUT_S=900` - fail/refund jobs that exceed this runtime.

## 2) Required hosted provider env vars (RunComfy)

- `HDRI_REMOTE_PROVIDER=runcomfy`
- `RUNCOMFY_API_TOKEN`
- `RUNCOMFY_DEPLOYMENT_ID`

Optional:

- `RUNCOMFY_BASE_URL=https://api.runcomfy.net`
- `RUNCOMFY_WORKFLOW_JSON_PATH=...` (send workflow JSON on each request)
- `RUNCOMFY_POLL_TIMEOUT_S=900`
- `RUNCOMFY_OVERRIDES_JSON={...}` (base node overrides)
- `RUNCOMFY_IMAGE_NODE_IDS`, `RUNCOMFY_PROMPT_NODE_IDS`, `RUNCOMFY_NEGATIVE_PROMPT_NODE_IDS`
- `RUNCOMFY_SEED_NODE_IDS`, `RUNCOMFY_STRENGTH_NODE_IDS`, `RUNCOMFY_REFERENCE_COVERAGE_NODE_IDS`
- `RUNCOMFY_DIMENSION_NODE_IDS`, `RUNCOMFY_STEPS_NODE_IDS`

These node ID mappings allow generic request fields (`image_b64`, prompts, seed, strength, reference coverage, width/height, quality steps) to map into your workflow graph without code changes.

## 3) Reverse proxy recommendations

- Put FastAPI behind Nginx/Caddy/Cloudflare Tunnel.
- Enforce HTTPS and request size limits suitable for base64 image uploads.
- Keep auth header forwarding enabled (`Authorization`, `X-Admin-Token`).
- Set timeout budget above expected job submission latency.
- Disable directory listing for static paths and only expose API routes.

## 4) Account and API key provisioning

Create accounts using:

- `POST /v1/accounts` with `X-Admin-Token: <HDRI_ADMIN_TOKEN>`
- Body:
  - `account_id` (optional; autogenerated if omitted)
  - `initial_tokens` (integer)

Response includes:

- `account_id`
- `api_key` (shown once; store securely)
- `tokens_remaining`

## 5) Token accounting defaults

- `quality_mode=fast` -> 1 token
- `quality_mode=balanced` -> 1 token
- `quality_mode=high` -> 2 tokens

Flow:

1. Tokens are reserved when `POST /v1/jobs/hdri` succeeds.
2. On success, reservation is consumed.
3. On failure/cancel/stale timeout, tokens are refunded automatically.

## 6) Operational checklist

- Configure monitoring for HTTP 5xx and job failure rate.
- Back up SQLite (`state.sqlite3`) or move to managed DB when needed.
- Rotate provider API token and admin token periodically.
- Validate RunComfy node-ID mapping after workflow edits.
